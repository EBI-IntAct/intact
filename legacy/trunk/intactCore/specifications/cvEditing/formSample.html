Specification for IntAct Cv Editing form:

The above form is a sample form for the IntAct editing tool for controlled vocabularies. The layout is not mandatory, it should just indicate that a "nice" layout should be used.

Additional comments:

  1. The two parts of the screen are largely independent, they might be implemented as subsequent forms.
  2. User identification should be based on the database login/password.
  3. On submit the form contents should be verified, and errors should be noted in an additional field on top of the form, on a red background.
  4. Javascript can be used to improve the usability of the form, but should not be essential. Example: It might be possible to define a good "tabbing" order of the fields in Javascript.
  5. The Session ID should allow to retrieve the current session in case of a server or client failure. It should be stored as a cooky on the client, but also be transferable from client to client by using it in a "retrieve session" form (after appropriate login). This obviously implies conservation of state on the server.
  6. The number of objects in a CvObject class will normally be in the order of 10, but may be up to 1000.
  7. The number of subelements (comments, crossreferences) will be low, in the order of 10.
Extensions:
  1. The form above is suitable for all current CvObjects, as they don't encode any specific attributes. However, this may change in the future, different form may be needed for different CvObject classes.
  2. Currently all CvObject classes are just lists, but some will become hierarchies in the future.
