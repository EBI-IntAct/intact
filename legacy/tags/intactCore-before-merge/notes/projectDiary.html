2002-04-12, hhe: Relational mapping of polymorphic reference.
The the IntAct core model, there is a relationship from Substrate to Interactor. Interactor is an abstract class. There is no direct way to support polymorphic references in the relational model.

I'll try to turn Interactor in a concrete class, and map it and all it's subclasses to the same table. This should allow an efficient implementation, at the only cost of unused columns in the table.

2002-04-11, hhe: Oracle implementation: CLOB or LONG?
The sequence is a large object, which can exceed Oracle's 4K limit for VARCHAR2. I've decided to use the CLOB data type because it's
  • is the Oracle-recommended type for large objects,
  • offers better direct access to the data than LONG,
  • it's the EBI standard to use CLOB for sequences.
Thanks to Phil McNeil, who provided lots of info to base the decision on.
