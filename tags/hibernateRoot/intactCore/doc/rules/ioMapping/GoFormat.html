Mapping rules for GO flat files

This document describes the mapping between GO format flat files and the IntAct database.

The GO flat file format is described at http://www.geneontology.org/doc/GO.doc.html#datarep

GO def file:

GO definition files are mapped into children of the abstact IntAct class CvObject.

GO tag
 IntAct attribute in CvObject
term
 fullName


goid
 Xrefs. CvDatabase.shortlabel="GO"
Xrefs.primaryid=goid
Xrefs.secondaryid=term (potentially truncated)
definition
 Annotation.CvTopic.shortlabel="Definition"
 
comment
 Annotation.CvTopic.shortlabel="Comment"
 
definition_reference
 Only mapped if the reference is to a pubmed id.
Xrefs. CvDatabase.shortlabel="PubMed"
Xrefs.CvXrefQualifier="GO-definition-ref"
Xrefs.primaryid=(parsed PMID)
Xrefs.secondaryid not mapped.


Tags not part of GO flat file format:
shortlabel
 shortLabel: If no shortLabel is defined, part of the term is used as the shortLabel. If term is too long, a unique label is generated from the goid.
intact_ac
 ac
[annotation topic]
 any IntAct annotation topic is mapped to a GO tag, the annotation content to the GO value.

GO dag file:

GO dag files are mapped into children of the abstact IntAct class CvDagObject.
