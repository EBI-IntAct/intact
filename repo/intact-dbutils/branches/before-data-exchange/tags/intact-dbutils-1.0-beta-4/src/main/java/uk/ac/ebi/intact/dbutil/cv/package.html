Handling of IntAct Controlled Vocabulary - Support for both .def/.dag and .obo files - Both parsers will use a Common data model that will be used then create/update the IntAct CVs. 
.def/.dag format:
----------------

    Documentation: http://www.geneontology.org/GO.format.shtml#goflat

    one IntAct CV can come as a single .def file (non hierarchical CV)
    one IntAct CV can come as a .def + a .dag file (hierarchical CV)

OBO format:
----------

    Documentation: http://www.geneontology.org/GO.format.shtml#oboflat

    The whole of the IntAct CVs come as a single obo file.
    note: The root of the OBO hierarchy is PPI (Protein Protein Interaction)
          Every node under the root is the root of a single IntAct CVs
          We will maintain a Mapping of MI terms to IntAct CV, including the fact that some PSI-MI CVs are
          not supported yet in IntAct
